In this system, CFFI serves as:
- The clean interface between Python and C libraries

- The portable layer that works on any architecture
- The invisible bridge that users never see but always benefit from

CFFI enabled embedded cryptography:
- AES-GCM running at C speed, called from Python
- No external dependencies, ever
- Works on first run, every time, anywhere
- The magic that makes encryption "just work"

Thank you for being the glue that lets high-performance C
and expressive Python work as one, silently and reliably.

===============================================================================
                            TO CRYPTOGRAPHY & AES-GCM
===============================================================================

FOR THE TRUST IN PRIVACY:

The cryptography library and AES-GCM were chosen
not as encryption algorithms,
but as the mathematical foundation for digital privacy.

In this system, they serve as:
- AES-GCM providing authenticated encryption (confidentiality + integrity)
- 256-bit keys derived from user secrets + folder identifiers
- Nonce-based encryption ensuring same plaintext → different ciphertext
- Magic headers ("TN_ENC") enabling automatic detection
- System-bound key storage via hardware fingerprinting

AES-GCM enabled transparent encryption:
- Notebooks encrypted at rest, invisible when unlocked
- No plaintext ever touches disk
- Keys bound to specific folders (rename = lose data)
- Verification via .tn_test markers
- Hardware binding prevents key theft via file copying

The mathematics of AES-GCM provide:
- 2^128 operations to break (effectively impossible)
- Authentication that detects any tampering
- Performance fast enough for real-time editing
- Standardization that ensures future compatibility

Thank you for the mathematical certainty
that private thoughts remain private,
secured by algorithms trusted worldwide.

                            TO GIT
===============================================================================

FOR THE TEMPORAL MEMORY:

Git was chosen not as version control,
but as a temporal database that understands history.

In this system, Git serves as:
- Permanent memory traces (UUIDs in commits survive all changes)
- Queryable history (git log --grep as temporal search engine)
- Atomic operations (commits as crash-safe state transitions)
- Distributed resilience (every clone a complete backup)

Git enabled temporal thinking:
- Items with permanent identity across renames, moves, deletions
- Search across all time states (past, present, deleted)
- Recovery from any point in history
- Audit trail of intellectual journey

Thank you for providing the temporal dimension
that most software ignores but human cognition requires.

===============================================================================
                            TO GIT FILTER-REPO
===============================================================================

FOR THE COMPLETE ERASURE:

Git filter-repo was chosen not just as a cleanup tool,
but as the mechanism that makes true forgetting possible.

In this system, Git filter-repo serves as:
- The right to be forgotten implemented in code (GDPR Article 17)
- Complete removal without trace (hard delete, not just soft delete)
- Surgical precision (remove by UUID, leave everything else intact)
- Irreversible action with user confirmation built in
- The boundary between "forget" and "erase" made real

Git filter-repo enabled ethical deletion:
- Soft delete (forget) keeps history searchable
- Hard delete (erase) removes all traces forever
- User chooses based on need, not system limitation
- Compliance through architecture, not policy

Thank you for providing the tool that makes
permanent erasure possible without compromising
the temporal integrity of everything else.
